游戏化 Gamification 笔记2

Playing a game is the voluntary attempt to overcome unecessary obstacles — Bernard Suits

User Progression 用户进程

归纳-最初的活动被简化，以帮助个人学习基本的机制、规则和目标。例如，减少或限制选择——只有一条路可走，几乎没有选择。
沉浸-增加难度和奖励之间的距离，以降低退出率(太容易或太难)
完成/掌握-可能是有限的(故事结束)或无限的(不同的组合)用户进程

Users role: the ‘Player’

  • Players are empowered to make choices and control outcomes.
  • To make the user a player

Fun elements 有趣的元素

  • 解决问题：问题的提出激发了内在的解决问题的欲望
  • 分享及合作活动
  • 探索、发现
  • 收集

Give participants one item ‘free’ to start with, and they are more likely to continue that if they start completely ‘empty’ set
Providing a Pay-off or Reward：
为了使效果最大化，奖励应该是:正确的 立即的
包括一些随机因素，例如，所期望的行为是奖励的2 - 3倍，而不是每次适量——足够但不要太多。
注意，对回报的期望不能取代参与活动的内在回报。

Game Architecture
叙述故事进程的使用有助于完整地组织
并将个体的活动沉浸到游戏的概念中
据说有七种基本的叙述方式。
通常反复出现的修辞结构会使读者通读整个故事。

Maps/Boards
表现层将个人提升到游戏范式中。个人帮助的参考点说明了他们来自哪里和他们需要去哪里。信息表示结构，或作为辅助表示映射，通常位于主信息表示窗口的一侧。

How？

  • Align real-world goals with the game
    • Specific, discrete
  • Structure the game
    • Narrative
    • Map/Board
  • Rules
    • What choices do players have?
    • Complexity vs simplicity
  • Structure the gaming mechanisms
    • competition
    • sharing
    • Progression and levels
  • Select and structure rewards
    • Real or virtual? Balance between getting started and completion
      将现实世界的目标与游戏结合起来，具体的、离散的结构，游戏叙述，地图/板，规则，玩家有什么选择?复杂性与简单，构建博弈机制，竞争，分享，进展和水平选择并组织，奖励真实还是虚拟?开始和完成之间的平衡
