创建Hexo博客记录

瞎倒腾了几天hexo，今天终于成功上线可以访问了。作为完全不懂代码的小白，只能参考网上文档各种试错，但有些情况还是不太一样。这里整理一些有用的经验，以便以后参考。btw使用的是Mac系统。

大致流程

安装Node.js →

官网链接：点击访问， 下载 MacOS 版本进行安装
然后运行：
sudo npm i -g npm
来安装Npm，npm 是 node 的包管理器

安装Git →

如果安装过 X-Code 的话，git 是包括在里面的
可以直接执行命令行安装：
brew install git

安装和初始化Hexo →

  • 在Terminal执行以下代码：
    npm install -g hexo-cli

    -g 用以全局安装

  • 创建一个装博客文件的空文件夹，有两种方法：

    1. 自己手动创建，然后在Terminal输入代码进入路径：
      cd 文件夹名字/
    2. 在Terminal输入代码创建文件夹
      mkdir 文件夹名字
      然后cd进入路径

  • 在这个路径开始 Hexo 的初始化：
    hexo init

本地测试运行 →

启动 Hexo 的本地服务器：
hexo server
hexo s 也可

访问本地 Hexo 博客：
不要关闭终端，在浏览器输入：localhost:4000

只有一篇 Hello World 文章，这是 Hexo 初始化的，里面包含了基础的操作

停止运行本地服务器：Ctrl+C

清理：
hexo clean

本地服务器预览：
hexo g
hexo d
hexo s

创建新文章：
hexo new 标题
文章在_posts 当中，是md格式的文件，在第5行代码 “—” 之后开始编辑文章内容

生成文章：
hexo generate

部署 →

注册github或coding
创建pages仓库

个性化域名 →

首先买域名
域名解析处添加三条记录：

  • to create an A record, point your apex domain to the IP addresses for GitHub Pages.
    185.199.108.153
    185.199.109.153
    185.199.110.153
    185.199.111.153

接下来在Terminal操作:
cd 你的hexo本地目录
cd source
创建CNAME文件：
touch CNAME
打开并编辑：
open CNAME
在打开的记事本文件填入域名
这里填www的话以后访问就必须输入www才行，不填就加不加都可以访问

回到你的hexo本地目录:
cd ../
重新生成部署:
hexo g -d

安装主题 →

现用主题Chic
修改主题在配置文件里更改代码
装复杂的主题插件有问题
目前解决不了这些bug

修改主题内容 →

menu

  • 菜单配置包括三个部分，第一是菜单项（名称和链接），第二是菜单项的显示文本，第三是菜单项对应的图标
  • 在设置 菜单项的名称和链接 中的名称并不会直接显示在网页上，而是会通过主题目录下的 languages/{language}.yml 找到对应的显示文本。

生成菜单文件夹:
hexo new page tags

将 source/tags/index.md 的内容修改为如下：

---
date: 2017-07-10 16:36:26
type: "tags"
comments: false
---
